## 🛠️ Code Quality Standards
For SDE-3/Tech Lead reviews, simple "leetcode style" code is insufficient. Solutions in this repo adhere to:
1. **Clean Code Principles**: Self-documenting variable/method names, minimal nesting, and Single Responsibility.
2. **Defensive Programming**: Explicit handling of edge cases (null inputs, overflows, empty bounds) at the entry point.
3. **Type Safety & Generics**: Utilizing modern Java standard libraries correctly and avoiding raw types.
4. **Performance Annotations**: Clear inline explanations of GC pressure (object allocations) and memory layouts.

---

## 🚀 Daily Execution Checklist

Every day, dedicate **1 hour** using the following rhythm:
1. **Understand (10 min)**: Read the problem, map edge cases, write test inputs on paper.
2. **Implement (20 min)**: Code the optimal solution in Java locally. Ensure it compiles and handles boundaries.
3. **Analyze & Document (20 min)**: Fill in the problem-level `README.md` explaining core intuition and system trade-offs.
4. **Git Sync (10 min)**: Commit changes using semantic commit messages and push to GitHub.
